Challenge 5 — Windows Security
Privilege Escalation 1 Frage · 1 richtige Antwort
Ein Engineer hat auf seinem Windows-Rechner einen Service installiert, der im SYSTEM-Kontext läuft. Er hat vergessen, die Berechtigungen einzuschränken — jeder Benutzer kann den Service stoppen und neu starten.

Der Pfad zur Service-Binary ist wie folgt konfiguriert:
Binary Path
C:\Program Files\CorpNet Helper\service.exe
Was kann ein Angreifer mit einem normalen Benutzeraccount in dieser Situation tun?
A
Er kann sich zu SYSTEM hochprivilegieren — indem er eine eigene Datei an einem Pfad ablegt, den Windows aufgrund des fehlenden Anführungszeichens zuerst prüft, und den Service dann neu startet.
✓ Korrekt. Ohne Anführungszeichen interpretiert Windows den Leerzeichen-Pfad als mögliche Trennpunkte und sucht zuerst nach C:\Program.exe, dann C:\Program Files\CorpNet.exe. Wer dort eine eigene Binary ablegen und den Service neu starten kann, führt Code als SYSTEM aus.
B
Er kann den Service über das Netzwerk direkt angreifen, ohne sich am System einloggen zu müssen.
✗ Falsch. Diese Schwachstelle erfordert lokalen Zugriff. Ein Angreifer braucht mindestens einen normalen Benutzeraccount auf dem Rechner.
C
Er kann den Service nur stoppen und damit kurzfristig einen Ausfall verursachen — mehr ist damit nicht möglich.
✗ Falsch. Das Stoppen allein wäre nur ein DoS. Der entscheidende Punkt ist der Unquoted Service Path: er ermöglicht in Kombination mit den Start-Rechten eine vollständige Privilege Escalation zu SYSTEM.
D
Es gibt kein reales Risiko, da der Angreifer ohnehin schon lokalen Zugriff benötigt und daher keine Eskalation stattfindet.
✗ Falsch. Lokaler Zugriff bedeutet nicht automatisch SYSTEM-Rechte. Genau diese Lücke — von Low-Privilege-User zu SYSTEM — ist der klassische Privilege-Escalation-Vektor.
Hint — 1 / 1
Schau dir den Pfad genau an: C:\Program Files\CorpNet Helper\service.exe

Ohne Anführungszeichen liest Windows Leerzeichen als mögliche Trennpunkte. Es versucht zuerst C:\Program.exe auszuführen, dann C:\Program Files\CorpNet.exe.

Wer an einer dieser Stellen eine eigene Datei ablegen kann — und den Service neu starten darf — führt Code als SYSTEM aus.
