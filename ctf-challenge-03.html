Vulnerability Report
CVE-2024-CN-0047 HIGH Stored XSS in Internal Ticketing System
A stored cross-site scripting vulnerability was discovered in the CorpNet internal helpdesk application (HelpDesk Pro 3.6.1). An unauthenticated attacker can submit a malicious support ticket containing a crafted JavaScript payload in the ticket body field. When a logged-in support agent opens the ticket, the script executes in their browser session and silently exfiltrates their session token to an attacker-controlled endpoint.

The attacker can then use the stolen session token to fully impersonate the support agent, gaining access to all tickets, internal user data, and administrative functions. No direct damage to system availability has been observed, but confidentiality and integrity are both critically affected. The vulnerability requires no prior authentication and no special privileges to exploit.
